Code

public enum Code : Int, CaseIterable

D1 error codes.

Note

each D1Error returned by D1 SDK contains one of these error codes.

Since

1.0.0

    The operation has been cancelled.

  • notLoggedIn

    The user is not logged in.

    Note

    Login before performing subsequent requests.

    Declaration

    Swift

    case notLoggedIn = 10005
  • notAuthorized

    The user is not authorized to perform this operation.

    Note

    Retry the operation again.

    Declaration

    Swift

    case notAuthorized = 10006
  • deviceEnvironmentUnsafe

    The device environment is potentially unsafe.

    Note

    Ensure that the device environment is safe.

    Declaration

    Swift

    case deviceEnvironmentUnsafe = 10007
  • core

    Error encountered whilst executing core operations.

    Note

    Integrator should check the underlying errors for more details. Ensure that the device has internet connection.

    Declaration

    Swift

    case core = 20001
  • risk

    Risk assessment failed.

    Category - Risk API

    Note

    Integrator should check the underlying errors for more details. Ensure risk credentials are correct and the device has internet connection.

    Remark

    D1Task.Components.riskURLString, D1Task.Components.riskClientID

    Declaration

    Swift

    case risk = 30001
  • card

    Errors encountered whilst executing card operations.

    Category - Card API

    Note

    Integrator should check the underlying errors for more details. Ensure card configurations are correct and the device has internet connection.

    Remark

    D1TaskComponents.digitalCardURLString, D1TaskComponents.issuerID

    Declaration

    Swift

    case card = 40001
  • cardNotSupported

    Device does not support making payment. It could be because of variety of reasons, e.g. hardware functionality, parental control restriction.

    Category - Card API

    Note

    Integrator should display information to user on why it is not supported.

    Declaration

    Swift

    case cardNotSupported = 40002
  • cardNoPendingIDV

    No digital card found that requires activation. No digital card is found with CardDigitizationState equal to CardDigitizationState.pendingIDVLocal or CardDigitizationState.pendingIDVRemote.

    Category - Card API

    Note

    Integrator may check the state by calling D1Task.cardDigitizationState(_:completion:).

    Swift
