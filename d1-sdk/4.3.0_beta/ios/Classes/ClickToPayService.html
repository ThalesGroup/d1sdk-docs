ClickToPayService

public final class ClickToPayService

Main interface of the Click to Pay service.

Use this service to access Click to Pay functionality including enrollment, opt-out, profile management, and card updates. This service provides async/await based APIs for better integration with modern Swift code.

  • enrol(_:consumerInfo:name:billingAddress:) Asynchronous

    Enroll the card in Click to Pay service. Once the enrollment is successful, the tokenization process is completed. The enrollment status in the integrator system will be updated via a push notification.

    Before calling this Click to Pay API, the integrator should display the default values of ConsumerInfo and BillingAddress and collect the end user inputs for ConsumerInfo.

  • profiles() Asynchronous

    This function retrieves the profile information including consumer details and the associated cards enrolled in the Click to Pay service for a specific customer.

    For end users who have not enrolled their cards in the Click to Pay service, an empty list will be returned by the API.

    Throws:

    • D1Error.Code.notLoggedIn
    • D1Error.Code.notAuthorized
    • D1Error.Code.deviceEnvironmentUnsafe
    • D1Error.Code.clickToPay
    • D1Error.Code.invalidArgument
    • D1Error.Code.cardNotManagedByD1
    • D1Error.Code.notInitialized

    Since

    4.3.0

    Declaration

    Swift

    public func profiles() async throws -> ClickToPayService.ProfileResult

    Return Value

    • Success: ProfileResult is returned with an array of Profile objects and an empty errorMessage.
    • Partial error: When two or more cards are enrolled in both Visa and Mastercard schemes (where one fails, the other succeeds), ProfileResult is returned with the Profile objects from successful enrollment scheme, together with errorMessage from failed enrollment scheme.
