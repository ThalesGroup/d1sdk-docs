  • D1Task

    Main class for D1 SDK use cases.

    Note

    It is recommended to initialize D1Task as soon as the application is launched or the earliest possible time.

    Note

    Risk API contains session ID internally. The session ID is used by backend to identify the user’s activity. The lifecycle of session ID is linked with D1Task lifecycle, i.e. one D1Task instance use same session ID for all the use cases.
